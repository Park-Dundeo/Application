# 사람 세움 프레임

> "관계로 세우고, 때가 되면 보낸다."

---

## 0. 한 줄 정의

**신앙 언어**: 부르심에 따라 사람 곁에 있으며, 그가 스스로 설 때까지 동행하다가, 때가 되면 기쁨으로 보내는 사명 단위.

**보편 언어**: 의미와 관계를 중심에 두고, 사람이 독립적으로 설 수 있도록 동행하다가 손을 떼는 프로젝트 단위.

핵심: **확장(크기)**이 아니라 **사람이 서는 것(독립)**.

---

## 1. 프로젝트 유형 3가지

모든 소명/관계/루틴에 동일한 프레임을 적용하면 관계가 관리 대상이 된다.
유형에 따라 적용 수준을 다르게 한다.

| 유형 | 설명 | 헌장 수준 | 시트 탭 |
|------|------|----------|---------|
| `routine` | 의심 없이 반복할 것들 — 판단보다 순종 | 없음 | 전체 현황만 |
| `relation` | 관계와 의무를 지속하는 것 — 방향보다 신실함 | 부르심 + 경계 + 회고 | 간소화 탭 |
| `mission` | 분별이 필요한 것 — 방향, 열매, 이양 | 풀 헌장 | 풀 탭 |

> 루틴에 Gate를 붙이지 않는다. 관계에 KPI를 붙이지 않는다.

---

## 2. 프레임 구성 요소 (mission 유형 기준)

### 1) 부르심

- **질문**: "나는 누구를, 어떤 방향으로 세우기 위해 여기 있는가?"
- 이 답이 흐릿하면 프로젝트를 시작하지 않는다.

### 2) 분별

- **3점 삼각형**: 기도(구함) + 공동체/조언자 확인 + 증거 점검
- 분별은 감정 안정이 아니라 **의사결정 게이트**다.

### 3) 동행

- **질문**: "이것은 정보 전달인가, 사람을 빚는 동행인가?"
- 프로그램이 아니라 "함께 있음"의 시간에서 형성된다.

### 4) 실천 루틴

- "잘하는 사람 1명"이 아니라 **반복 가능한 포맷 1개**를 만든다.
- 내가 없어도 돌아가는 구조를 처음부터 의식한다.

### 5) 사람이 서는 증거

- 숫자 하나가 아니라 **증거 2개**: 질적(변화 사례) + 양적(재참여/독립 지표)
- "열심"이 아니라 "사람이 서는 것"이 증명이다.

### 6) 보냄과 이양

- **질문**: "이 구조는 사람을 붙잡는가, 보내는가?"
- 이양이 최종 성공이다. 내가 없어도 됨이 목표다.

### 7) 회고와 회복

- 실패는 종료가 아니라 **안정화 → 교정 → 재위임** 경로다.
- 회고 없이는 동행이 관성이 된다.

### 8) 경계

- 경계 4종 (최소): 시간 상한 / 돈 상한 / 역할 상한 / 가치 상한
- 경계 없는 헌신은 번아웃과 우상화로 이어진다.

---

## 3. Gate 3단계

### Gate A — 시작 (Seed)

- 부르심 1문장 + 최소 실행 루틴 1개 + 경계 2개
- **질문**: "이 시즌에 이것을 시작하는 게 맞는가?"
- A를 통과해야 B로 간다.

### Gate B — 심화 (Deepen)

- 열매 증거 1개 이상 + 실천 루틴 작동 + 동행 대상 명확화
- **질문**: "더 깊이 들어가는 것이 순종인가, 지금을 지키는 것이 순종인가?"
- B는 확장이 아니다. 더 깊어지거나, 지금 자리를 신실하게 지키는 것.

### Gate C — 이양/마침 (Handoff)

- 리더(사람) + 매뉴얼(문서) + 본질 3가지 + 점검 리듬
- **질문**: "이제 보내는 게 맞는가? 형태를 바꿔야 하는가? 멈추는 게 맞는가?"

---

## 4. 종료/전환 프로토콜 — R.E.S.T.

- **R (Review)**: 처음 부르심이 살아있는가?
- **E (Evidence)**: 사람이 서고 있는 증거가 있는가?
- **S (Shepherd decision)**: 심화 / 안정 / 중단·전환 중 하나 선택
- **T (Transfer)**: 이양(사람·자산·자료) + 기록 + 축복으로 마침

---

## 5. 유형별 헌장 템플릿

### routine 헌장 (없음)
그냥 한다. 분별보다 순종이 먼저인 것들.

---

### relation 헌장 (3요소)
```
1. 부르심 (1문장): 이 관계/의무를 지키는 이유
2. 경계: 시간 / 돈 / 역할 / 가치
3. 회고 리듬: 언제 점검하는가
```

---

### mission 헌장 (10요소)
```
1. 부르심/미션 (1문장)
2. 분별 방식 (기도 + 공동체)
3. 동행의 형태
4. 실천 루틴 (1개 이상)
5. 사람이 서는 증거 (질적 + 양적)
6. 보냄/이양 단계
7. 회고 리듬
8. 경계 4종
9. 이양 조건
10. 종료/전환 조건
```

---

## 6. 함정: 이 프레임이 다단계처럼 느껴질 때

**재생산이 목표가 되는 순간** 프레임이 왜곡된다.
아래를 점검한다:

- "더 많이"를 원하고 있는가? → 부르심으로 돌아가라
- 사람을 수단으로 보기 시작했는가? → Gate C로 가거나 멈춰라
- 내가 없으면 안 된다는 느낌이 드는가? → 이양 설계가 없는 것이다

**핵심 메커니즘은 유입이 아니라 보냄이다.**
성공의 증거는 네트워크 크기가 아니라 **내가 없어도 서는 사람**이다.
